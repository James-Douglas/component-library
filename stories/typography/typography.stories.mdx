# CTM Typography Guidelines

* The Manor global component handles the base html tags, styling them by default. Include `<Manor/>` at the top of your app tree to use the styling.
* For non html elements/styling, such as `subtitle` or `microcopy`, you can use the relevant styled component, or add the class to the element.

## Headings: `(h1-h6)`

Global styles handles html tags to have the required style, meaning you can write your markup as you would usually.

## Body Text ( *primary and secondary* )
For body text, there are several ways to achieve the desired styling. You can use the `<BodyText>` component,
give it a variant of either `primary` or `secondary`. Or, you can use a regular `p` tag to get the primary variant.
Using a p tag relies on manor global styles and additionally, the `<BodyText>` component will default to the primary variant.

The class `body-secondary` also exists for other use cases.

<Preview>
  <Story name="body-primary">
    <DemoContainer>
      <Overline>Use case: Homepage, Landing pages, Journey Headings</Overline>
      <Overline>variant: primary</Overline>
      <BodyText variant='primary'>Compare The Market</BodyText>
      <Overline>variant: secondary </Overline>
      <BodyText variant='secondary'>Compare The Market</BodyText>
    </DemoContainer>
  </Story>
</Preview>

## Subtitles ( *primary and secondary* )

The `<Subtitle>` component provides the desired styling, has `primary` and `secondary` variants. The component will default to the primary variant.

Also available as a classes: `.subtitle-primary` and `.subtitle-primary`
